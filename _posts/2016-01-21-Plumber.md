Simply use [gulp-plumber](https://www.npmjs.com/package/gulp-plumber) to fix the pipe when using Gulp. Gulp-Plumber lets you handle errors more elegantly and avoid interrupting any of your Gulp tasks running in the background.

The typical scenario is that the pipe "breaks" and Gulp stops running when error is thrown. This leaves you having to correct the error and execute `gulp serve` again. It can get frustrating at some point having to run the command multiple times unnecessarily.
